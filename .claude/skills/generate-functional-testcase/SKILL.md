# Skill: 根据需求文档生成测试用例

## 触发方式

当用户提供需求文档（Markdown 格式）并要求生成测试用例时触发。用户可以：
- 直接粘贴需求文档内容
- 提供需求文档的文件路径
- 使用 `/生成测试用例` 命令

## 输入

用户需要提供：
1. **需求文档**：Markdown 格式的需求描述（必须）
2. **测试类型**（可选，默认生成全部）：
   - `功能测试` — 验证功能是否符合需求
   - `接口测试` — 验证接口输入输出
   - `UI测试` — 验证页面交互和展示
3. **输出格式**（可选，默认 Markdown）：
   - `markdown` — 输出 `.md` 文件
   - `excel` — 输出 `.xlsx` 文件（需要 openpyxl）

---

## 执行步骤

### 第一步：解析需求文档

仔细阅读需求文档，提取以下信息：
- **功能模块**：文档描述的核心功能点
- **业务规则**：明确的约束条件、边界值、校验规则
- **用户角色**：涉及哪些角色（普通用户、管理员等）
- **前置依赖**：功能依赖的数据或状态
- **接口信息**：如有接口描述，提取请求/响应结构
- **UI交互**：如有页面描述，提取交互逻辑

### 第二步：设计测试场景

基于解析结果，按以下维度设计测试场景：

**功能测试场景**：
- 正向流程（Happy Path）：标准操作流程，预期成功
- 边界值：最大值、最小值、临界值
- 异常流程：非法输入、缺少必填项、权限不足
- 业务规则验证：每条业务规则至少一个用例

**接口测试场景**（如需求包含接口描述）：
- 正常请求：合法参数，验证响应结构和数据
- 参数校验：必填参数缺失、参数类型错误、参数越界
- 权限校验：未登录、无权限角色
- 幂等性：重复请求的处理

**UI测试场景**（如需求包含页面描述）：
- 页面加载：元素是否正确展示
- 交互操作：点击、输入、提交等
- 表单校验：前端校验规则
- 响应反馈：成功/失败提示

### 第三步：生成测试用例

每个测试用例包含以下字段：

| 字段 | 说明 |
|------|------|
| 用例ID | 格式：`TC-{模块缩写}-{序号}` 如 `TC-LOGIN-001` |
| 用例标题 | 简洁描述测试目的，格式：`[场景] 验证 [功能点] [条件]` |
| 测试类型 | 功能测试 / 接口测试 / UI测试 |
| 优先级 | P0（核心流程）/ P1（重要功能）/ P2（边界异常）/ P3（低频场景） |
| 前置条件 | 执行用例前需要满足的状态或数据 |
| 测试步骤 | 编号的操作步骤，每步一行，具体可执行 |
| 预期结果 | 每个步骤对应的预期结果，与步骤一一对应 |

**优先级判断规则**：
- P0：核心业务流程，系统不可用则阻塞上线
- P1：重要功能，影响主要用户体验
- P2：边界条件、异常处理，影响部分场景
- P3：低频使用、兼容性、细节优化

### 第四步：输出测试用例

#### Markdown 格式输出

将测试用例写入文件，路径为：`testcases/{模块名}/testcases.md`

文件结构：
```markdown
# {模块名} 测试用例

> 需求来源：{需求文档标题或描述}
> 生成时间：{当前日期}
> 用例总数：{数量}

## 功能测试用例

### TC-{模块}-001：{用例标题}

- **优先级**：P0
- **前置条件**：
  1. 条件一
  2. 条件二
- **测试步骤**：
  1. 步骤一
  2. 步骤二
- **预期结果**：
  1. 步骤一预期结果
  2. 步骤二预期结果

---

## 接口测试用例

（同上格式）

## UI测试用例

（同上格式）
```

#### Excel 格式输出（当用户要求时）

使用 Python 脚本生成 Excel 文件：

```python

from openpyxl.styles import Font, PatternFill, Alignment, Border, Side

```

### 第五步：输出摘要

生成完成后，输出：
- 生成的文件路径
- 用例统计：各类型用例数量、各优先级数量
- 覆盖的功能点列表

## 示例

**输入需求片段**：
```
## 用户登录功能
用户通过手机号+密码登录系统。
- 手机号格式：11位数字，1开头
- 密码：6-20位，包含字母和数字
- 连续失败5次锁定账号30分钟
- 登录成功跳转首页，显示用户昵称
```

**生成用例示例**：

```
TC-LOGIN-001：[正向] 验证合法手机号和密码登录成功
- 优先级：P0
- 前置条件：
  1. 系统正常运行
  2. 已注册手机号 13800138000，密码 abc123456
- 测试步骤：
  1. 打开登录页面
  2. 输入手机号 13800138000
  3. 输入密码 abc123456
  4. 点击登录按钮
- 预期结果：
  1. 登录页面正常显示
  2. 手机号输入框显示 13800138000
  3. 密码输入框显示掩码
  4. 跳转至首页，顶部显示用户昵称

TC-LOGIN-002：[异常] 验证连续失败5次后账号被锁定
- 优先级：P1
- 前置条件：
  1. 已注册手机号 13800138000
  2. 账号当前未被锁定
- 测试步骤：
  1. 使用错误密码登录，重复5次
  2. 第6次使用正确密码登录
- 预期结果：
  1. 前5次提示"密码错误"，第5次提示"账号已锁定，请30分钟后再试"
  2. 提示"账号已锁定，请30分钟后再试"，无法登录
```

---

## 注意事项

- 测试步骤要具体可执行，避免模糊描述（如"验证功能正常"）
- 预期结果要可量化、可验证，避免主观判断
- 边界值用例要包含边界内、边界上、边界外三种情况
- 如需求文档描述不清晰，在生成用例前先列出疑问点，询问用户确认
- 用例ID在同一文档内保持唯一且连续
